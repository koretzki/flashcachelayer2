Flash Cache Layer (FCL)

The FCL is a core software for hybrid storage systems that integrate high-performance SSDs into large-capacity HDDs, or MLC based SSDs. The main role of the FCL is to manage SSDs as a data cache with various cache algorithms. It is implemented in CMU DiskSim with MSR SSD Extension.

Various FCLs

  • FP-FCL (Fixed Partitioning): SSD is used as a data cache device with LRU algorithm.
  • RW-FCL (Read Write Partitioning): Read and Write data would be distinguished due to GC of SSD. (Now implementing)
  • OP-FCL (Optimal Partitioning): Our proposed scheme in USENIX FAST 2012. (Now implementing)

Supported Feature

  • LRU-based Data Caching
  • Background Data Migration
  • Prefetching (not yet implemented)
  • Trim Command (not yet implemented)

Installation

  • download
  • tar xvfj fcl-beta-release-0.1.tar.gz
  • cd fcl-beta-release-0.1
  • make
  • cd hybrid_valid/maxtor-sample1
  • download http://www.mpi-sws.org/~vahldiek/diskmodels/maxtor146g.tar.gz
  • tar xvfj maxtor146g.tar.gz
  • mv maxtor146g/{{{*}}} hybrid_valid/maxtor-sample1
  • set the parameter Max Queue Length = 1 in maxtor146g.diskspecs file
  • ./script_fixed
  • ./script_rw
  • ./script_optimal
  • ./response
